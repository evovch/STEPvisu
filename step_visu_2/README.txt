Чтобы начать работать с solution'ом, нужно прописать системную переменную EXTSOFT, указывающую на папку, где есть glew, glm, flex-bison и т.д.
см. скриншот.
Также должна быть системная переменная QTDIR указывающая на папку с Qt5.
В моём случае это "C:\Qt\Qt5.4.2_x86\5.4\msvc2013_opengl"

В соответствии с задумкой программного продукта требуются следующие блоки:

parsing
stepentities
brepentities
representations
graphics
gui

папка CommentRemoval нужна для препроцессинга входных файлов. Однако в какой-то момент нужно преложить усилия к тому, чтобы избавиться от препроцессинга. На данный момент он возможен только под Linux.

run_scan.sh - скрипт для запуска тестирования программного продукта на наборе входных файлов, работает только под Linux.

QT pro используется только в Linux. По сути дела сейчас это замена файлу проекта студии.

wrapper'ы используются только в Windows.


Для реализации триангуляции требуется, чтобы каждый объект типа линия имел в себе аппроксимирующий полигон.
Для реализации построения полигона кривой см. Голованов, 9.4, 1.5, (1.6)
Для реализации триангуляции см. Голованов, 9.4, 1.7 (1.8), 9.7
Также для общего сведения см. Голованов, 4.2

Build under Linux:

./build.sh

Prereq:
flex, bison, glm
